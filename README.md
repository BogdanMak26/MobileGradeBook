# 📱 GradeBook Mobile — Flutter App

Мобільний застосунок електронного журналу для **Військового інституту телекомунікацій та інформатизації імені Героїв Крут**.

Розроблено як мобільна частина системи GradeBook — підключається до існуючого REST API сервера.

---

## 🎯 Про проєкт

Система GradeBook вже функціонує як веб-застосунок. Цей репозиторій — її мобільна версія на Flutter, яка повністю повторює функціонал веб-клієнта та додає підтримку **офлайн-режиму** — критично важливу функцію в умовах нестабільного мережевого покриття.

## 🏗️ Архітектура

Застосунок побудований за патерном **MVVM** з чітким розподілом відповідальності:

```
lib/
├── core/
│   ├── api/              # Dio HTTP клієнт з auth interceptor + репозиторії
│   ├── auth/             # Keycloak OAuth2 PKCE авторизація
│   ├── local/            # Офлайн-кеш (SharedPreferences) + черга операцій
│   ├── notifications/    # Сервіс сповіщень + модель налаштувань
│   ├── network/          # Моніторинг стану мережі
│   ├── sync/             # Авто-синхронізація при відновленні з'єднання
│   └── mock/             # Мокові дані для розробки UI
├── features/
│   ├── auth/             # Вхід (View + ViewModel)
│   ├── dashboard/        # Головна сторінка
│   ├── disciplines/      # Дисципліни
│   ├── grades/           # Журнал оцінок + оцінки курсанта
│   ├── analytics/        # Рейтинг групи
│   ├── schedule/         # Розклад
│   ├── profile/          # Профіль
│   ├── notifications/    # Налаштування сповіщень
│   ├── groups/           # Моя група
│   └── admin/            # Адмін-панель
└── shared/
    ├── theme/            # Кольори та стилі (відповідають веб-версії)
    └── widgets/          # Спільні компоненти (connectivity overlay, sync chip)
```

## 🔄 Offline-First стратегія

Ключова особливість застосунку — повноцінна робота без інтернету:

- **Читання** — дані кешуються локально при кожному успішному запиті (`LocalCache`)
- **Запис** — операції (оцінки, відвідуваність) потрапляють у `OfflineQueue` при відсутності мережі
- **Синхронізація** — `SyncService` автоматично виконує всі операції з черги при відновленні з'єднання
- **Індикація** — `ConnectivityOverlay` показує банер офлайн-режиму та тост при поверненні онлайн
- **Лічильник** — `SyncStatusChip` відображає кількість операцій в очікуванні

## 🔔 Система сповіщень

Рольові push-сповіщення з тонким налаштуванням:

| Роль | Категорії сповіщень |
|---|---|
| **Курсант** | Важливі заняття (залік/іспит/КР) з нагадуванням за 1/24/48 год, нові та низькі оцінки, зміни розкладу, оголошення кафедри |
| **Викладач** | Незаповнені журнали, нагадування перед заняттям (15/30/60 хв), зміни розкладу, нові курсанти |
| **Начальник кафедри** | Тижневий/місячний дайджест успішності, критичні показники (<60%), прострочені журнали |
| **Адмін / Навч. відділ** | Нові реєстрації, помилки синхронізації, тижневі звіти, зміни в структурі |

Налаштування зберігаються локально (SharedPreferences) і застосовуються одразу.

## 🔐 Авторизація

Застосунок використовує Keycloak з **OAuth2 Authorization Code Flow + PKCE**:

- Немає `client_secret` (public client — безпечно для мобільних)
- Токени зберігаються в `flutter_secure_storage` (Android Keystore / iOS Keychain)
- Авторефреш токенів через Dio interceptor
- Підтримка всіх ролей: Курсант, Викладач, Начальник кафедри, Навчальний відділ, Суперадмін
- Deep link callback: `com.viti.gradebook://callback`

## 🚀 Запуск

```bash
# 1. Встановити залежності
flutter pub get

# 2. Запустити
flutter run
```

> **Примітка:** Сервер `gradebook.viti.edu.ua` захищений Cloudflare Zero Trust.
> Для підключення до реального API потрібен CF Service Token (`CF-Access-Client-Id` + `CF-Access-Client-Secret`), який додається в `lib/core/api/api_client.dart`.
> До отримання токена застосунок працює на мокових даних.
